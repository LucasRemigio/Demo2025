SPMS - Serviços Partilhados do Ministério da Saúde, EPE
Atualmente, o processo de actualização dos dados no Portal da
Transparência envolve a receção via email de dados pelas instituições de
saúde, validação da estrutura e fiabilidade dos dados e, quando
necessário, transformações de forma a adequar os novos dados aos que
estão publicados no portal. Dado que este processo de validação manual
está sujeito a erros, e dado o aumento no número de indicadores no portal,
é necessário automatizar o processo de receção, processamento e
integração dos dados.
Como tal, começámos recentemente a desenvolver um Back Office para o
Portal da Transparência, de forma a:
- Permitir a submissão de dados pelas instituições no back office;
- Automatizar o processo de validação, verificando se os novos dados
respeitam a estrutura e se são coerentes com os dados existentes no
portal;
- Alertar o responsável pela submissão dos dados de possíveis erros de
validação, de forma a que os mesmos possam ser imediatamente corrigidos
e novamente submetidos;
- Criar serviços web para atualizar automaticamente o Portal da
Transparência com os dados validados.
Tecnologias: De forma a construir o back office, foi dada uma arquitetura
com base noutros sistemas da SPMS, de forma a adaptar a arquitetura base
ao back office da transparência. Estou neste momento desenvolver a partir
dessa arquitetura base e, para tal, tenho estado a aprender como construir
aplicações web com Java 8, usando a web framework do Apache Wicket e
a framework Hibernate para mapeamento dos objectos Java com as
entidades numa
base
de dados. Tenho aprofundado os
meus
conhecimentos em SQL e gestão de base de dados Oracle de forma a criar
o repositório para o back office. Sendo o meu primeiro projeto de
desenvolvimento web, estou a ter contacto pela primeira vez com as
típicas ferramentas de web - HTML, CSS e JavaScript.

Monitor SNS - Desenvolvimento em C# com .NET Core (desde setembro
2017):
SPMS - Serviços Partilhados do Ministério da Saúde, EPE
Em paralelo com as operações no Portal da Transparência, tive a
oportunidade de ajudar um colega no desenvolvimento do Monitor SNS.
O Monitor SNS permite o envio automático de dados a cada instituição de
saúde (hospitais e centros de saúde), com dados referentes a indicadores
de qualidade e eficiência nos serviços prestados pela instituição,
permitindo que a mesma evolua no sentido de melhorar o desempenho
face aos indicadores.
O projeto envolveu o desenvolvimento inicial da aplicação, onde tive a
oportunidade de aprender com o meu colega a programar com C# e com
ferramentas .NET Core. Desde então, tenho sido responsável pelo
melhoramento contínuo da aplicação, bem como da integração de novos
dados para envio automático às instituições.
Tecnologias: Monitor SNS foi desenvolvido em C# com a web framework
.NET Core, e com a Entity Framework para mapeamento dos objectos C#
com entidades numa base de dados SQL Server.
